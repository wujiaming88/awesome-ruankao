## 试题一（必做题）

### 题目描述

某大型金融科技公司拟构建一个**智能风控决策平台**，该平台整合多源数据（交易流水、用户行为、征信数据、设备指纹等），通过规则引擎和机器学习模型实时评估交易风险，并做出放行、拦截或人工审核的决策。平台需支持每日亿级交易量的实时风控处理。

在系统需求分析阶段，项目组整理了以下需求：

（a）系统在接收到交易请求后，应在200毫秒内完成风险评估并返回决策结果。  
（b）系统应支持风控规则的热更新，新规则发布后无需重启服务即可生效，且新规则从编写到上线应在2小时内完成。  
（c）当风控引擎节点发生故障时，系统应在5秒内自动将请求切换到备用节点，确保风控服务不中断。  
（d）系统应对所有风控决策过程进行完整的审计日志记录，包括输入数据、触发规则、模型评分和最终决策。  
（e）系统应采用数据脱敏和字段级加密技术，确保用户敏感信息（身份证号、银行卡号）在传输和存储过程中的安全。  
（f）在交易高峰期（如"双十一"），系统应支持自动水平扩展，峰值处理能力应达到日常的5倍以上。  
（g）系统应支持A/B测试功能，允许同时运行多个版本的风控模型，按流量比例进行灰度发布。  
（h）当检测到大规模欺诈攻击时（如同一IP在1分钟内发起超过100笔交易），系统应在3秒内触发自动熔断并告警。  
（i）系统应支持多种数据源的接入（内部交易系统、外部征信机构、设备指纹服务），且新数据源的接入应在3个工作日内完成。  
（j）风控决策结果应同时推送到交易系统、客服系统和监管报送系统，推送延迟不超过500毫秒。  
（k）系统应提供可视化的规则编辑器，业务人员无需编程即可配置风控规则。  
（l）系统应支持对历史交易进行回溯分析，1亿条交易记录的批量回溯应在4小时内完成。

## 试题二

### 题目描述

某电商平台的订单系统日均处理订单量超过500万笔，数据库采用MySQL 8.0。随着业务增长，订单表（orders）的单表数据量已超过8000万行，相关的订单明细表（order_items）数据量超过3亿行。系统出现以下问题：

1. 订单查询响应时间从原来的50ms上升到800ms以上
2. 大促期间数据库CPU使用率持续95%以上，频繁出现慢查询
3. 订单状态更新的并发冲突导致数据不一致问题
4. 月底财务对账时的批量统计查询严重影响在线业务性能

目前订单表结构如下：
```sql
CREATE TABLE orders (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_no VARCHAR(32) NOT NULL,
    user_id BIGINT NOT NULL,
    shop_id BIGINT NOT NULL,
    total_amount DECIMAL(10,2),
    status TINYINT DEFAULT 0,
    pay_time DATETIME,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_create_time (create_time)
);
```

项目组李工提出采用**分库分表+CQRS（命令查询职责分离）**方案进行优化。

### 问题1（10分）

（1）请设计订单表的分库分表方案，说明分片键（Sharding Key）的选择及理由。（5分）

（2）请说明采用`user_id`作为分片键时，如何高效实现"按商家（shop_id）查询订单列表"的需求？给出至少两种方案并对比优缺点。（5分）

### 问题2（8分）

请说明CQRS架构模式的核心思想，并设计本系统中CQRS的具体实现方案，解决"在线查询"与"批量统计"互相干扰的问题。

### 问题3（7分）

针对订单状态更新的并发冲突问题，请分别说明**乐观锁**和**悲观锁**的实现方式，并分析各自适用的场景，给出本系统推荐采用的方案及理由。

## 试题三

### 题目描述

某互联网公司构建了一个**实时数据处理平台**，采用Apache Kafka作为消息中间件，连接上游的业务系统和下游的数据分析系统。平台每天处理超过50亿条消息，峰值吞吐量达到每秒200万条消息。

在系统运行过程中，团队遇到了以下问题：

1. 消费者组（Consumer Group）在高峰期频繁触发Rebalance，导致消费延迟剧增
2. 部分Topic的Partition数据量严重不均衡（数据倾斜），个别Partition的积压量是其他Partition的10倍以上
3. 在消费端采用"至少一次"（At-Least-Once）语义时，下游系统出现重复数据处理的问题
4. 消息从生产到被消费的端到端延迟在高峰期从50ms上升到5秒以上

### 问题1（10分）

（1）请分析Kafka消费者组频繁触发Rebalance的可能原因（至少列举4种），并给出相应的解决方案。（6分）

（2）请说明Kafka Partition数据倾斜的原因，并给出至少2种解决方案。（4分）

### 问题2（8分）

（1）请分别解释Kafka的三种消息传递语义：At-Most-Once、At-Least-Once、Exactly-Once，并说明各自的实现机制。（4分）

（2）在"至少一次"语义下，请给出消费端实现**幂等性处理**的具体方案（至少2种）。（4分）

### 问题3（7分）

针对端到端延迟上升的问题，请从Producer、Broker、Consumer三个环节分别分析可能的瓶颈，并给出对应的优化策略。

## 试题四

### 题目描述

某知识管理公司拟构建一个**基于大模型的RAG（检索增强生成）智能问答系统**，面向企业客户提供基于私有知识库的智能问答服务。系统需要支持多种文档格式（PDF、Word、PPT、网页等）的知识导入，通过语义检索和大语言模型生成高质量的回答，并支持多轮对话。

系统核心流程如下：
1. **知识导入阶段**：文档解析 → 文本分块（Chunking）→ 向量化（Embedding）→ 存储到向量数据库
2. **查询阶段**：用户问题 → 问题改写/扩展 → 向量检索 + 关键词检索（混合检索）→ 重排序（Reranking）→ 上下文组装 → LLM生成回答
3. **评估与优化**：答案质量评估 → 用户反馈收集 → 检索策略调优

在架构评审中，团队围绕以下关键问题展开了讨论：
- 如何选择合适的文本分块策略以保证检索质量？
- 向量检索和关键词检索如何融合以提升召回率？
- 如何评估RAG系统的整体效果并持续优化？

### 问题1（10分）

（1）请说明RAG系统中文本分块（Chunking）的常见策略（至少3种），并分析各策略的适用场景和优缺点。（6分）

（2）请说明为什么需要在向量检索之外引入关键词检索（即混合检索），并设计混合检索的融合方案。（4分）

### 问题2（8分）

请设计该RAG系统的整体技术架构，说明各核心模块的职责和关键技术选型，并画出（或用文字描述）系统架构图。

### 问题3（7分）

请说明如何评估RAG系统的效果，列举关键评估指标（至少4个），并给出持续优化的策略。

## 试题五

### 题目描述

某政府机构拟构建一个**零信任安全架构**的电子政务系统，该系统服务于多个政府部门，处理包含公民个人信息、行政审批数据等敏感信息。系统需要在开放网络环境中为不同部门的工作人员提供安全的远程办公访问。

传统的基于网络边界的安全架构（VPN+防火墙）已无法满足安全需求：
1. 远程办公人员通过各种网络环境访问系统，传统VPN一旦接入即获得内网全部权限
2. 内部人员的横向移动攻击难以防范
3. 多部门协作时的细粒度权限控制困难
4. 移动设备和个人设备（BYOD）的安全管控不足

### 问题1（10分）

（1）请说明零信任安全架构的核心原则（至少4条）。（4分）

（2）请设计该电子政务系统的零信任安全架构，说明核心组件及其职责。（6分）

### 问题2（8分）

（1）请说明零信任架构中"持续验证"（Continuous Verification）的实现机制。（4分）

（2）请对比零信任架构与传统VPN+防火墙架构在安全性、用户体验、运维复杂度方面的差异。（4分）

### 问题3（7分）

在该系统中，如何实现基于**属性的访问控制（ABAC）**模型？请说明ABAC模型的核心要素，并结合电子政务场景给出具体示例。
