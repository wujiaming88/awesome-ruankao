## 七、英语阅读理解（第71-75题）

**阅读材料：**

The Retrieval-Augmented Generation (RAG) paradigm has emerged as a transformative approach to addressing the limitations of Large Language Models (LLMs). While LLMs demonstrate remarkable capabilities in natural language understanding and generation, they often （71） from outdated knowledge, hallucinations, and inability to access domain-specific information.

RAG systems integrate an external knowledge retrieval component with the generation capabilities of LLMs. The typical RAG pipeline consists of three main stages: indexing, retrieval, and generation. During the indexing stage, documents are split into chunks and converted into vector （72） using embedding models, which are then stored in a vector database. When a user submits a query, the retrieval stage identifies the most （73） document chunks by computing similarity between the query vector and stored vectors.

The retrieved context is then combined with the original query to form an augmented prompt, which is fed to the LLM for response generation. This approach allows the model to ground its responses in factual, up-to-date information rather than relying （74） on its parametric knowledge.

Recent advances in RAG include multi-modal retrieval, adaptive chunking strategies, and re-ranking mechanisms that further improve retrieval （75）. These developments make RAG an essential architecture pattern for enterprise AI applications that require accuracy and trustworthiness.
