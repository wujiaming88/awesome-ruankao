# 系统架构设计师备考指南

> 适用于 2024-2026 年考试，基于最新机考改革后的考试形式

## 📋 考试概览

| 项目 | 内容 |
|------|------|
| 考试级别 | 高级 |
| 考试科目 | 综合知识 + 案例分析 + 论文 |
| 考试形式 | 机考（2024年起） |
| 考试时间 | 每年5月和11月各一次 |
| 合格标准 | 三科均 ≥ 45分（满分75分） |
| 备考周期 | 建议 3-6 个月 |

## 🎯 科目分析

### 科目一：综合知识（150分钟）

- **题型**: 75道单选题，每题1分
- **范围**: 计算机基础、软件工程、架构设计、数据库、网络、安全、法规等
- **特点**: 
  - 知识面极广，覆盖计算机科学各领域
  - 约30%为往年真题变形
  - 5道英语阅读题（通常较简单）
  - 2024年起增加了AI、大模型等新兴技术考点

**备考策略**:
1. 通读教材 1-2 遍，建立知识框架
2. 刷近5年真题（至少3遍）
3. 重点掌握：架构风格、设计模式、质量属性、UML
4. 薄弱环节专项突破

### 科目二：案例分析（90分钟）

- **题型**: 5道大题选做3题（2024年起变为5选3）
- **范围**: 架构设计、数据库设计、嵌入式系统、Web应用等
- **特点**:
  - 必答题通常为架构质量属性分析
  - 近年常考：缓存策略、微服务、分布式系统
  - 新增考点：ROS2、大语言模型架构

**备考策略**:
1. 掌握质量属性六要素（必考）
2. 熟练掌握常见架构风格及其优缺点
3. 练习数据库范式判定和SQL优化
4. 了解嵌入式基础概念

### 科目三：论文（120分钟）

- **题型**: 4选1，包含摘要（300-330字）+ 正文（2000-2500字）
- **范围**: 架构设计方法论、质量属性、开发过程等
- **特点**:
  - 需要结合实际项目经验
  - 字数要求严格（机考有字数统计）
  - 评分标准：论点正确、论据充分、结构清晰

**备考策略**:
1. 准备 3-4 个万能项目案例
2. 掌握五段式论文结构
3. 背诵各主题的核心理论点
4. 考前至少手写练习 5 篇

## 📅 3个月备考计划

### 第1个月：基础打底

| 周次 | 任务 | 重点 |
|------|------|------|
| 第1周 | 通读教材第1-5章 | 计算机基础、操作系统、网络 |
| 第2周 | 通读教材第6-10章 | 数据库、软件工程、面向对象 |
| 第3周 | 通读教材第11-15章 | 架构设计、质量属性、评估方法 |
| 第4周 | 通读教材第16-20章 | 安全、法规、新技术 |

### 第2个月：真题突破

| 周次 | 任务 | 目标 |
|------|------|------|
| 第5周 | 刷2022-2023年综合知识真题 | 正确率 > 60% |
| 第6周 | 刷2024年综合知识真题 | 正确率 > 65% |
| 第7周 | 练习案例分析真题 | 掌握答题模板 |
| 第8周 | 论文模板准备 + 练写1篇 | 掌握结构 |

### 第3个月：冲刺提高

| 周次 | 任务 | 目标 |
|------|------|------|
| 第9周 | 综合知识查缺补漏 | 正确率 > 70% |
| 第10周 | 案例分析专项训练 | 时间控制在90分钟内 |
| 第11周 | 论文练写2-3篇 | 字数和时间控制 |
| 第12周 | 模拟考试 + 考前复习 | 全流程演练 |

## 💡 高频考点

### 架构风格（必考）

| 风格 | 关键词 | 考频 |
|------|--------|------|
| 管道-过滤器 | 数据流、过滤器可并行 | ★★★★★ |
| 仓库/黑板 | 中央数据结构、知识源 | ★★★★★ |
| 分层架构 | 调用/返回、职责分离 | ★★★★☆ |
| MVC | 模型-视图-控制器 | ★★★★☆ |
| 微服务 | 独立部署、服务治理 | ★★★★☆ |
| 事件驱动 | 发布/订阅、松耦合 | ★★★☆☆ |

### 设计模式（必考）

- **创建型**（5种）: 工厂方法、抽象工厂、单例、建造者、原型
- **结构型**（7种）: 适配器、桥接、组合、装饰、外观、享元、代理
- **行为型**（11种）: 策略、模板方法、观察者、迭代器、职责链、命令、备忘录、状态、访问者、中介者、解释器

### 质量属性（案例必考）

| 属性 | 描述 | 常见战术 |
|------|------|----------|
| 性能 | 响应时间、吞吐量 | 缓存、负载均衡 |
| 可用性 | 正常运行时间占比 | 冗余、心跳检测、故障转移 |
| 安全性 | 防止未授权访问 | 认证、授权、加密 |
| 可修改性 | 适应变化的能力 | 模块化、信息隐藏 |
| 可测试性 | 验证正确性的难度 | 接口规范、依赖注入 |

## 🔧 机考技巧

1. **时间分配**: 综合知识每题不超过2分钟，不会就跳过
2. **标记功能**: 善用标记按钮，先做会的再回头
3. **论文打字**: 提前练习打字速度，目标 2500字/60分钟
4. **草稿纸**: 机考只提供有限草稿纸，计算题要精打细算
5. **复制粘贴**: 论文中重复引用的项目背景可以复制粘贴
