## 综合知识速记（选择题保分）

### 操作系统 🔴
- **进程三态**：就绪⇄执行→阻塞→就绪（阻塞不能直接→执行）
- **死锁四条件**：互斥+请求保持+不剥夺+循环等待（**不能破坏互斥**）
- **不死锁最少资源**：n×(k-1)+1（n个进程各需k个资源）
- Cache对程序员**透明**；多道程序=并发≠并行（单CPU）

### 安全 🔴
- **四属性**：机密性(不泄露)、完整性(不篡改)、可用性(能用)、不可否认性(防抵赖)
- **对称加密**：AES/DES/3DES/SM4（加解密同一密钥）
- **非对称**：RSA/ECC/SM2（公钥加密私钥解密）
- AES-256：14轮加密，256位密钥，抗量子128位有效强度

### 软件工程 🔴
- **开发模型**：瀑布→增量→螺旋(原型+风险)→喷泉(面向对象)→RUP(用例驱动+架构中心+迭代增量)
- **测试覆盖强度**：语句<判定<条件<判定/条件<条件组合<路径（**条件覆盖不一定满足判定覆盖**）
- **维护四类型**：改正性(修BUG)→适应性(变环境)→完善性(加功能)→预防性(防未来)
- **需求工程**：发现→分类→协商→文档化（迭代过程）
- **敏捷**：以人为中心（非用例），迭代增量，少文档

### 数据库 🔴
- **范式链**：1NF→2NF(消除部分依赖)→3NF(消除传递依赖)→BCNF→4NF(消除多值依赖)
- **三级模式**：外模式(用户视图)→概念模式(整体逻辑)→内模式(物理存储)
- **SQL**：HAVING与GROUP BY连用；关系运算不含删除；自然连接去重复列；笛卡尔积=m×n

### 架构风格 🔴（⭐每年必考2-3题）
- **五大类**：数据流(管道-过滤器) / 调用返回(层次) / 独立构件(事件驱动) / 仓库(黑板) / 虚拟机(解释器)
- **架构风格** = 词汇表 + 约束
- **管道-过滤器**：数据单向流动，过滤器独立，**可以并行**
- **解释器四组件**：解释器引擎 + 被解释程序 + 引擎内部状态 + 程序当前状态
- **黑板**：中央数据结构，不确定性问题求解
- **4+1视图**：逻辑+进程(运行状态)+开发+物理(软硬件映射)+场景
- **B/S三层**（不是二层C/S特殊形式）；ADL语言：UniCon/Rapide/Wright/ACME

### 质量属性与评估 🔴（⭐案例必答基础）
- **运行期**：性能/可用性/安全性/可伸缩性/互操作性
- **开发期**：可维护性/可扩展性/可重用性/可测试性/可移植性
- **ATAM**：效用树分类，最初关注性能/安全性/可修改性/可用性
- **SAAM**：最早形成文档的架构评估方法

### 设计模式 🔴
- **创建型5**：工厂方法/抽象工厂/单例/原型/建造者
- **结构型7**：适配器/桥接/组合/装饰器/外观/享元/代理（⚠️桥接不是创建型）
- **行为型11**：策略/模板/观察者/状态/命令/中介者/备忘录/迭代器/访问者/职责链/解释器

### UML 🔴
- **用例关系**：包含(include)/扩展(extend)/泛化（**不含聚合**）
- **序列图三消息**：同步(实心箭头)/异步(开放箭头)/返回(虚线箭头)

### 专利权 🔴
- **客体**：发明+实用新型+外观设计（**不含商标**）
- 保护范围以**权利要求书**为准

### Redis缓存 🔴
- **RDB**：二进制快照，恢复快，可能丢数据
- **AOF**：命令日志，数据安全(最多丢1秒)，恢复慢
- **生产建议**：同时开启RDB+AOF
- **全量复制**：PSYNC→BGSAVE→发RDB→发缓冲区命令
- **缓存击穿**：热点key过期 → 互斥锁(强一致) 或 逻辑过期+异步更新(高可用)

### Lambda架构 🔴
- **三层**：批处理层(准确性)+速度层(补偿延迟)+服务层(合并视图)
- **vs Kappa**：Kappa仅流处理一条链路，通过重放保证一致性

### 论文写作 🔴
- 每次固定搭配：1题架构+1题测试/质量+1题数据+1题新技术
- SOA/微服务/EDA/Serverless 必须会写一个

## 案例必答题速记（⭐考前必看）

### 质量属性场景六要素（连续4次必考！）

```
刺激源 → 刺激 → 环境 → 制品 → 响应 → 响应度量
(谁)    (什么事) (什么条件) (哪部分) (怎么做) (怎么衡量)
```

### 需求→质量属性映射口诀

| 看到这些词 → | 填这个属性 |
|------------|-----------|
| X秒/分钟内、响应时间、吞吐量 | **性能** |
| 故障切换、不间断、自动恢复 | **可用性** |
| 防止未授权、加密、安全 | **安全性** |
| X天内完成、扩容、新模块 | **可修改性** |
| 多语言、快捷键、适配分辨率 | **易用性** |
| 自动化测试框架 | **可测试性** |
| 多终端、第三方对接 | **互操作性** |

> ⚠️ 有**时间约束**就是**性能**（"1分钟内导入"=性能，不是功能性）
> ⚠️ "X天内完成开发"是**可修改性**（不是性能，性能指运行时）

## 论文结构速记（⭐提笔前默念）

```
摘要 → 150字（项目+角色+主题+效果）

正文 2200-2500字：
  一、项目背景   300-400字  行业+规模+痛点+我是架构师
  二、理论阐述   500-600字  定义→分类→特点→对比（答第2问）
  三、项目实践   900-1100字 选型→实施→问题→解决（答第3问）⭐最重要
  四、效果总结   300-400字  量化效果+不足+展望

结尾 → 100字（反思+改进方向）
```

**论文铁律**：
- ✅ 角色=系统架构设计师（不是开发/测试/PM）
- ✅ 效果必须量化（提升X倍/降低X%/缩短X天）
- ✅ 必须写问题+解决方案（至少2-3个）
- ✅ 字数2200-2500（少了扣分，超2800也扣分）
- ❌ 不能只写理论或只写实践，必须结合

## 高频易错点（最后扫一眼）

1. 条件覆盖 **不一定** 满足判定覆盖
2. 管道-过滤器 **可以** 并行（不是不能并行）
3. B/S 是三层架构（**不是**二层C/S特殊形式）
4. 桥接模式是 **结构型**（不是创建型）
5. 用例关系 **不含聚合**（包含/扩展/泛化）
6. 关系运算 **不含删除**
7. 死锁预防 **不能破坏互斥** 条件
8. Cache 对程序员 **透明**
9. SAAM 是最早的架构评估方法（不是ATAM）
10. 心跳比Ping/Echo更优（被监控方主动发送，资源消耗低）
