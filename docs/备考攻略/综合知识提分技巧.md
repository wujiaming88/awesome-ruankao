## 二、时间分配策略

### 总体原则

- **平均每题 2 分钟**，但不同类型题目分配不同
- **不要在单题上花超过 3 分钟**
- **先易后难，标记跳过**

### 推荐时间分配

| 阶段 | 时间 | 题目范围 | 策略 |
|------|------|----------|------|
| 第一轮 | 90 分钟 | 全部 75 题 | 会的直接做，不确定的标记跳过 |
| 第二轮 | 40 分钟 | 标记的题目 | 仔细分析，排除法 |
| 第三轮 | 15 分钟 | 剩余未答题 | 蒙题，确保全部填写 |
| 检查 | 5 分钟 | 检查答题卡 | 确认无遗漏、无误选 |

### 各题型建议用时

| 题型 | 数量（约） | 单题用时 | 总用时 |
|------|-----------|---------|--------|
| 秒杀题（记忆型） | 20-25 题 | 30-60 秒 | 15-20 分钟 |
| 分析题（理解型） | 25-30 题 | 1.5-2 分钟 | 40-50 分钟 |
| 计算题 | 5-8 题 | 2-3 分钟 | 15-20 分钟 |
| 英语题 | 5 题 | 2-3 分钟 | 10-15 分钟 |
| 难题/偏题 | 10-15 题 | 标记跳过 | 最后处理 |

## 四、计算题速解公式

### 1. 挣值管理（必考 ⭐⭐⭐⭐⭐）

```
SV = EV - PV          （进度偏差，正=提前）
CV = EV - AC          （成本偏差，正=节约）
SPI = EV / PV         （进度绩效指数，>1 好）
CPI = EV / AC         （成本绩效指数，>1 好）
EAC = BAC / CPI       （典型偏差下的完工估算）
EAC = AC + (BAC - EV) （非典型偏差下的完工估算）
ETC = EAC - AC        （完工尚需估算）
VAC = BAC - EAC       （完工偏差）
```

**速记**：EV 是核心，减法看偏差，除法看指数

### 2. 三点估算 / PERT

```
期望值 E = (O + 4M + P) / 6
标准差 σ = (P - O) / 6
方差 V = σ² = [(P - O) / 6]²
```

**关键路径方差** = 各关键活动方差之和  
**项目标准差** = √(关键路径方差)

**正态分布置信区间**：
- E ± 1σ → 68.26%
- E ± 2σ → 95.46%
- E ± 3σ → 99.73%

### 3. 沟通渠道

```
渠道数 = N(N-1) / 2
```

| N | 渠道数 |
|---|--------|
| 5 | 10 |
| 6 | 15 |
| 8 | 28 |
| 10 | 45 |
| 12 | 66 |
| 15 | 105 |

**变形题**：团队增加 1 人后增加多少渠道？→ 增加 N 条（N 为增加前的人数）

### 4. 决策树 / EMV

```
EMV = Σ(概率 × 收益)
选择 EMV 最大的方案
```

### 5. 可用性 / 可靠性

```
可用性 A = MTBF / (MTBF + MTTR)
```

| MTBF | MTTR | 可用性 |
|------|------|--------|
| 99h | 1h | 99% |
| 999h | 1h | 99.9% |

**串联系统可靠性** = R₁ × R₂ × ... × Rₙ  
**并联系统可靠性** = 1 - (1-R₁)(1-R₂)...(1-Rₙ)

### 6. 网络相关计算

```
子网主机数 = 2^n - 2  （n 为主机位数）
子网数 = 2^m         （m 为借用位数）
```

### 7. McCabe 环路复杂度

```
V(G) = E - N + 2
V(G) = 判定节点数 + 1
V(G) = 区域数（含外部区域）
```

## 六、蒙题技巧（最后手段）

> ⚠️ 蒙题只是保底策略，平时应扎实学习。以下技巧仅在完全不会时使用。

### 1. 排除法（最有效）

- 先排除明显错误的选项
- 排除 1 个 → 正确率从 25% 提升到 33%
- 排除 2 个 → 正确率提升到 50%

### 2. 常见排除规则

- **绝对化表述通常是错的**：「一定」「必须」「所有」「绝不」
- **含有「可能」「通常」「一般」的选项更可能正确**
- **两个选项意思相反，答案很可能在其中之一**
- **两个选项极其相似（只差一两个字），答案很可能在其中之一**

### 3. 统计经验

- 75 题中 ABCD 分布大致均匀（各约 18-19 题）
- 如果已确定的答案中某个选项明显偏少，剩余不确定的题可倾向选该选项
- **不建议全部蒙同一个字母**

### 4. 技术直觉

- 软件工程题偏向「规范化、系统化」的选项
- 安全题偏向「更安全、更严格」的选项
- 架构题偏向「松耦合、高内聚」的选项

### 5. 最重要的原则

> **千万不要空着不填！蒙对一题就是一分！**

## 七、错题分析方法

### 错题分类

| 类别 | 特征 | 应对策略 |
|------|------|----------|
| A类：知识盲区 | 完全不知道这个知识点 | 补充学习，添加 Anki 卡片 |
| B类：理解偏差 | 知道但理解错误 | 重新理解，记录易混淆点 |
| C类：审题不仔细 | 看错题、漏看条件 | 练习审题习惯，圈关键词 |
| D类：计算失误 | 公式对但算错 | 多练计算题，验算 |
| E类：超纲题 | 教材未覆盖的新内容 | 了解即可，不必深究 |

### 错题复习节奏

```
做错当天：立即分析，记录错因
第 2 天：重做错题（不看答案）
第 4 天：再次复习
第 7 天：第三次复习
第 15 天：最终确认
```

### 错题本模板

```markdown
## 错题 #001
**日期**: 2026-03-15
**题目**: [粘贴题目]
**我的答案**: B
**正确答案**: D
**知识点**: 架构风格 - 管道过滤器
**错因分类**: B类（理解偏差）
**错误原因**: 混淆了管道-过滤器和事件驱动的区别
**正确理解**: 管道-过滤器是数据流驱动，事件驱动是事件/消息触发
**关联知识**: 五大架构风格对比
**复习状态**: ✅ 已掌握
```

### 错题统计分析

每周统计错题分布：

```
知识域分布：
  架构设计: ████████ 8 题
  数据库:   █████ 5 题
  网络:     ████ 4 题
  安全:     ███ 3 题
  法规:     ██ 2 题

错因分布：
  A类（盲区）: ████████ 40%
  B类（偏差）: ██████ 30%
  C类（审题）: ████ 20%
  D类（计算）: ██ 10%
```

根据统计结果调整复习重点：**A 类多 → 补知识，C 类多 → 练审题**
