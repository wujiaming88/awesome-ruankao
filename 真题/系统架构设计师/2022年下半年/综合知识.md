### 第1题

**题目**: 云计算服务体系结构如下图所示，图中①、②、③分别与 SaaS PaaS IaaS 相对应，图中①、②、③应为( )

- A. 应用层、基础设施层、平台层

- C. 平台层、应用层、基础设施层
- D. 平台层、基础设施层、应用层

### 第2题

**题目**: 前趋图(Precedence Graph)是一个有向无环图，记为:→={(Pi，Pj)|Pi must complete before Pj may start}，假设系统中进程P={P1，P2，P3，P4，P5，P6，P7，P8}，且进程的前趋图如下图所示。那么，该前趋图可记为( )

- A. →={(P1，P2)，(P1，P3)，(P1，P4)，(P2，P5)，(P3，P5)，(P4，P7)，(P5，P6)，(P5，P7)，(P7，P6)，(P4，P5)，(P6，P7)，(P7，P8)}

### 第3题

**题目**: 若系统正在将( )文件修改的结果写回磁盘时系统发生掉电，则对系统影响相对较大。

- A. 系统目录
- B. 空闲块
- C. 用户程序
- D. 用户数据

**答案**: A

**解析**: 系统目录文件记录了文件系统的组织结构和文件索引信息，若写入时掉电导致目录损坏，可能导致整个文件系统无法正确访问，影响最大。用户数据和程序的损坏仅影响个别文件。

### 第4题

**题目**: 在磁盘调度管理中，应先进行移臂调度，再进行旋转调度。假设磁盘移动臂位于20号柱面上，进程的请求序列如下表所示。如果采用最短移臂调度算法，那么系统的响应序列应为( )

- A. ②⑧③④⑤①⑦⑥⑨
- B. ②③⑧④⑥⑨①⑤⑦
- C. ④⑥⑨⑤⑦①②⑧③
- D. ④⑥⑨⑤⑦①②③⑧

**答案**: C

**解析**: 最短寻道时间优先（SSTF）算法从当前磁头位置出发，每次选择距离当前磁头最近的磁道请求进行服务。从20号柱面开始，依次选择最近的请求，得到响应序列。

### 第5题

**题目**: 采用三级模式结构的数据库系统中，如果对一个表创建聚簇索引，那么改变的是数据库的( )

- A. 外模式
- B. 模式
- C. 内模式
- D. 用户模式

**答案**: C

**解析**: 数据库三级模式为外模式、模式（概念模式）、内模式（物理模式）。聚簇索引改变了数据的物理存储顺序，属于内模式的范畴。外模式是用户视图，模式是逻辑结构，内模式是物理存储结构。

### 第6题

**题目**: 假设系统中有正在运行的事务，若要转储全部数据库，则应采用( )方式。

- A. 静态全局转储
- B. 动态增量转储
- C. 静态增量转储
- D. 动态全局转储

**答案**: D

**解析**: 系统中有正在运行的事务，不能停止系统运行，因此不能使用静态转储（需系统停止运行），只能使用动态转储。题目要求转储全部数据库，所以是全局转储。因此选择动态全局转储。

### 第7题

**题目**: 给定关系模式R(U，F)，其中U为属性集，F是U上的一组函数依赖，那么函数依赖的公理系统(Armstrong公理系统)中的分解规则是指( )为F所蕴涵

- A. 若X→Y，Y→Z，则X→Z
- B. 若Y⊆X⊆U，则X→Y
- C. 若X→Y，Z⊆Y，则X→Z
- D. 若X→Y，Y→Z，则X→YZ

**答案**: C

**解析**: Armstrong公理系统包括三条基本规则：自反律（若Y⊆X，则X→Y）、增广律（若X→Y，则XZ→YZ）、传递律（若X→Y，Y→Z，则X→Z）。分解规则是由基本规则推导出的：若X→Y且Z⊆Y，则X→Z，即X能决定Y，则X也能决定Y的任意子集。选项A是传递律，B是自反律，D是合并规则。

- B. （略，含图）

### 第9题

**题目**: 以下关于鸿蒙操作系统的叙述中，不正确的是( )

- A. 鸿蒙操作系统整体架构采用分层的层次化设计，从下向上依次为：内核层、系统服务层、框架层和应用层
- B. 鸿蒙操作系统内核层采用宏内核设计，拥有更强的安全特性和低时延特点
- C. 鸿蒙操作系统架构采用了分布式设计理念，实现了分布式软总线、分布式设备虚拟化、分布式数据管理和分布式任务调度等四种分布式能力
- D. 架构的系统安全性主要体现在搭载HarmonyOS的分布式终端上，可以保证"正确的人，通过正确的设备，正确地使用数据"

**答案**: B

**解析**: HarmonyOS（鸿蒙操作系统）采用的是**微内核**设计，而非宏内核。微内核将最少的功能放在内核中，其他服务运行在用户态，具有更好的安全性和灵活性。宏内核是Linux、Windows等传统操作系统的设计方式。B选项将"微内核"错误说成了"宏内核"。

**题目**: GPU目前已广泛应用于各行各业，GPU中集成了同时运行在GHz的频率上的成千上万个core，可以并行处理大量数据。最新的GPU峰值性能可高达( )以上。

### 第12题

**题目**: 通常，嵌入式中间件没有统一的架构风格，根据应用对象的不同可存在多种类型，比较常见的是消息中间件和分布式对象中间件。以下有关消息中间件的描述中，不正确的是( )

- A. 消息中间件是消息传输过程中保存消息的一种容器
- B. 消息中间件具有两个基本特点：采用异步处理模式、应用程序和应用程序调用关系为松耦合关系
- C. 消息中间件主要由一组对象来提供系统服务，对象间能够跨平台通信
- D. 消息中间件的消息传递服务模型有点对点模型和发布-订阅模型之分

**答案**: C

**解析**: C选项描述的是**分布式对象中间件**（如CORBA）的特征，而非消息中间件。分布式对象中间件通过一组对象提供系统服务，支持对象间跨平台通信。消息中间件的核心特征是通过消息队列实现异步通信和松耦合，支持点对点和发布-订阅两种消息传递模型。

### 第16～17题

**题目**: 系统( 16 )是指在规定的时间内和规定条件下能有效地实现规定功能的能力。它不仅取决于规定的使用条件等因素，还与设计技术有关。常用的度量指标主要有故障率(或失效率)、平均失效等待时间、平均失效间隔时间和可靠度等。其中，( 17 )是系统在规定工作时间内无故障的概率。

**（第16题）选项**:
- A. 可靠性
- B. 可用性
- C. 可理解性
- D. 可测试性

**（第17题）选项**:
- A. 失效率
- B. 平均失效等待时间
- C. 平均失效间隔时间
- D. 可靠度

**答案**: 第16题 **A**，第17题 **D**

**解析**: 系统**可靠性**是指在规定时间内和规定条件下有效实现规定功能的能力，其度量指标包括故障率、MTTF、MTBF和可靠度。**可靠度**R(t)是系统在[0,t]时间内无故障运行的概率，是可靠性的核心量化指标。可用性关注的是系统可正常使用的时间比例。

### 第22题

**题目**: 与瀑布模型相比，( )降低了实现需求变更的成本，更容易得到客户对于已完成开发工作的反馈意见，并且客户可以更早地使用软件并从中获得价值。

- A. 快速原型模型
- B. 敏捷开发
- C. 增量式开发
- D. 智能模型

**答案**: C

**解析**: **增量式开发**将系统功能分成多个增量（版本），每个增量交付一部分可用功能。其优点是：降低需求变更成本（每次只开发部分功能）、客户能更早获得反馈（每个增量交付后即可使用评估）、客户更早获得价值（不必等整个系统完成）。快速原型模型主要用于需求确认，敏捷开发虽也有类似优点但更强调迭代和响应变化。

### 第27题

- D. 安全性

**答案**: 第56题 **A**，第57题 **C**

**解析**: 质量属性场景由六部分组成：**刺激源、刺激、环境、制品、响应、响应度量**。"想要学习系统特性、有效使用系统、使错误影响最低、适配系统、对系统满意"这些都是用户在使用系统时的期望和行为，属于**易用性（Usability）**质量属性的典型刺激。

### 第58题

**题目**: 改变加密级别可能会对安全性和性能产生非常重要的影响，因此在软件架构评估中，该设计决策是一个( )。

- A. 敏感点
- B. 风险点
- C. 权衡点
- D. 非风险点

**答案**: C

**解析**: 在ATAM（架构权衡分析方法）中：**敏感点**是指为实现某一质量属性而做出的架构决策；**权衡点**是影响**多个**质量属性的架构决策，是多个敏感点的交集；**风险点**是可能导致不良后果的决策。改变加密级别同时影响**安全性**和**性能**两个质量属性，因此是一个**权衡点**。

### 第59题

**题目**: 效用树是采用架构权衡分析方法(Architecture Tradeoff Analysis Method, ATAM)进行架构评估的工具之一，其树形结构从根部到叶子节点依次为( )。

- A. 树根、属性分类、优先级、质量属性场景
- B. 树根、质量属性、属性分类、质量属性场景
- C. 树根、优先级、质量属性、质量属性场景
- D. 树根、质量属性、属性分类、优先级

**答案**: B

**解析**: ATAM中效用树（Utility Tree）的层次结构从根到叶依次为：**树根（效用/Utility）**→ **质量属性**（如性能、可用性、安全性等）→ **属性分类/细化**（如性能下的响应时间、吞吐量等）→ **质量属性场景**（具体的、可度量的场景描述，标注优先级）。

### 第60题

**题目**: 平均失效等待时间(MTTF)和平均失效间隔时间(MTBF)是进行系统可靠性分析时的重要指标，在失效率为常数和修复时间很短的情况下，( )。

- A. MTTF远远小于MTBF
- B. MTTF和MTBF无法计算
- C. MTTF远远大于MTBF
- D. MTTF和MTBF几乎相等

**答案**: D

**解析**: MTBF = MTTF + MTTR，其中MTBF为平均故障间隔时间，MTTF为平均无故障时间，MTTR为平均修复时间。当**修复时间很短**（MTTR→0）时，MTBF ≈ MTTF，即两者几乎相等。在实际应用中，对于高可靠性系统，通常MTTR远小于MTTF，所以两者近似相等。

### 第61～62题

**题目**: 在进行软件系统安全性分析时，( 61 )保证信息不泄露给未授权的用户、实体或过程；完整性保证信息的完整和准确，防止信息被非法修改；( 62 )保证对信息的传播及内容具有控制的能力，防止为非法者所用。

**（第61题）选项**:
- A. 完整性
- B. 不可否认性
- C. 可控性
- D. 机密性

**（第62题）选项**:
- A. 完整性
- B. 安全审计
- C. 加密性
- D. 可控性

**答案**: 第61题 **D**，第62题 **D**

**解析**: 信息安全的基本属性：**机密性（Confidentiality）**保证信息不泄露给未授权的用户、实体或过程；**完整性（Integrity）**保证信息的完整和准确，防止被非法修改；**可控性（Controllability）**保证对信息的传播及内容具有控制能力，防止为非法者所用。此外还有可用性和不可否认性。

### 第63题

**题目**: 在进行架构评估时，首先要明确具体的质量目标，并以之作为判定该架构优劣的标准。为得出这些目标而采用的机制叫做场景，场景是从( )的角度对与系统的交互的简短描述。

- A. 用户
- B. 系统架构师
- C. 项目管理者
- D. 风险承担者

**答案**: D

**解析**: 在架构评估中，场景是从**风险承担者（Stakeholder，利益相关者）**的角度对系统交互的简短描述。利益相关者包括用户、开发者、管理者、运维人员等所有与系统有利害关系的人，他们各自关注不同的质量属性。不仅仅是用户视角，而是所有利益相关者的视角。

### 第64题

**题目**: 5G网络采用( )可将5G网络分割成多张虚拟网络，每个虚拟网络的接入、传输和核心网是逻辑独立的，任何一个虚拟网络发生故障都不会影响到其它虚拟网络。

- A. 网络切片技术
- B. 边缘计算技术
- C. 网络隔离技术
- D. 软件定义网络技术

**答案**: A

**解析**: **网络切片（Network Slicing）**是5G网络的核心技术之一，可以在同一物理网络基础设施上创建多个逻辑独立的虚拟网络。每个切片有独立的接入、传输和核心网，相互隔离，一个切片故障不影响其他切片。可以为不同应用场景（如eMBB、URLLC、mMTC）定制网络切片。

### 第65题

**题目**: 以下WiFi认证方式中，( )使用了AES加密算法，安全性更高。

- A. 开放式
- B. WPA
- C. WPA2
- D. WEP

**答案**: C

**解析**: **WPA2**采用**AES（Advanced Encryption Standard）**加密算法和CCMP协议，安全性最高。WEP使用RC4加密算法，已被证明存在严重安全漏洞。WPA作为过渡方案使用TKIP协议（仍基于RC4），安全性优于WEP但不如WPA2。开放式无加密认证。

### 第66题

**题目**: 程序员甲将其编写完成的某软件程序发给同事乙并进行讨论，之后甲放弃该程序并决定重新开发，后来乙将该程序稍加修改并署自己名在某技术论坛发布。以下说法中，正确的是( )

- A. 乙的行为侵犯了甲对该程序享有的软件著作权
- B. 乙行为未侵权，因其发布的场合是以交流学习为目的的技术论坛
- C. 乙的行为没有侵犯甲的软件著作权，因为甲已放弃该程序
- D. 乙对该程序进行了修改，因此乙享有该程序的软件著作权

**答案**: A

**解析**: 根据《计算机软件保护条例》，软件著作权自开发完成之日起自动产生，甲是该程序的著作权人。甲虽然"放弃该程序"（放弃继续开发），但**并未放弃著作权**。乙未经甲许可，将甲的程序修改后以自己名义发布，侵犯了甲的署名权、修改权和发布权（如果甲尚未发表则还侵犯了发表权）。

### 第67题

**题目**: 以下关于软件著作权产生时间的叙述中，正确的是( )

- A. 软件著作权产生自软件首次公开发表时
- B. 软件著作权产生自开发者有开发意图时
- C. 软件著作权产生自软件开发完成之日起
- D. 软件著作权产生自软件著作权登记时

**答案**: C

**解析**: 根据《计算机软件保护条例》第十四条规定，软件著作权自**软件开发完成之日**起产生，采用自动保护原则，无需登记或发表。著作权登记是自愿行为，不是权利产生的条件，但登记可以作为著作权的初步证明，有利于纠纷解决。

### 第68题

**题目**: M公司将其开发的某软件产品注册了商标，为确保公司可在市场竞争中占据优势地位，M公司对员工进行了保密约束，此情形下，该公司不享有( )。

- A. 软件著作权
- B. 专利权
- C. 商业秘密权
- D. 商标权

**答案**: B

**解析**: 题目中M公司：开发了软件（自动享有**软件著作权**）、注册了商标（享有**商标权**）、对员工进行保密约束（享有**商业秘密权**）。但题目未提到申请专利，且**专利权需要向国家知识产权局申请并获得授权**后才能享有，不是自动产生的。因此公司不享有专利权。

### 第69题

**题目**: 计算机产生的随机数大体上能在(0,1)区间内均匀分布。假设某初等函数f(x)在(0,1)区间内取值也在(0,1)区间内，如果由计算机产生的大量的(M个)随机数对(r1,r2)中，符合r2≤f(r1)条件的有N个，则N/M可作为( )的近似计算结果。

- A. 求解方程f(x)=x
- B. 求f(x)极大值
- C. 求f(x)的极小值
- D. 求积分

**答案**: D

**解析**: 这是**蒙特卡洛方法（Monte Carlo Method）**求定积分的经典应用。在单位正方形(0,1)×(0,1)中随机撒M个点，落在函数f(x)曲线下方（即r2≤f(r1)）的点数为N，则N/M近似等于f(x)在[0,1]区间上的定积分值∫₀¹f(x)dx，因为该积分值等于曲线下方面积与单位正方形面积之比。

### 第70题

**题目**: 某项目包括A、B、C、D四道工序，各道工序之间的衔接关系、正常进度下各工序所需的时间和直接费用、赶工进度下所需的时间和直接费用如下表所示。该项目每天需要的间接费用为4.5万元。根据此表，以最低成本完成该项目需要( )天。

- A. 7
- B. 9
- C. 10
- D. 5

**答案**: A

**解析**: 项目最优工期需要在直接费用增加和间接费用减少之间取得平衡。首先找出关键路径，然后逐步压缩关键路径上赶工成本最低的工序，计算每次压缩后的总成本（直接费用+间接费用），直到找到总成本最低的方案。根据题目数据计算，以最低成本完成项目需要7天。

### 第71～75题

**题目**: (英语阅读题) Micro-service is a software development technology, which advocates dividing a single application into a group of small services, which coordinates and cooperates with each other to provide ultimate value for users. The micro-service ( 71 ) has many important benefits. First, it solves the problem of business complexity. It decomposes the original huge single application into a group of services. Although the total amount of functions remains the same, the application has been decomposed into manageable services. The development speed of a single service is much faster, and it is easier to understand and ( 72 ). Second, this architecture allows each service to be ( 73 ) independently by a team. Developers are free to choose any appropriate technology. Third, the micro-service architecture mode enables each service to be ( 74 ) independently. Developers never need to coordinate the deployment of local changes to their services. These types of changes can be deployed immediately after testing. Finally, the micro-service architecture enables each service to ( 75 ) independently.

**（第71题）选项**:
- A. architecture
- B. software
- C. application
- D. technology

**（第72题）选项**:
- A. develop
- B. maintain
- C. utilized
- D. deploy

**（第73题）选项**:
- A. planned
- B. developed
- C. utilized
- D. deployed

**（第74题）选项**:
- A. utilized
- B. developed
- C. tested
- D. deployed

**（第75题）选项**:
- A. analyze
- B. use
- C. design
- D. expand

**答案**: 第71题 **A**，第72题 **B**，第73题 **B**，第74题 **D**，第75题 **D**

**解析**: 本题为微服务架构相关英语阅读理解。
- 第71题：micro-service **architecture**（微服务架构）是标准术语搭配。
